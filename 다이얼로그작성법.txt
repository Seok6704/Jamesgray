Json으로 다이얼로그를 구성하려면 다음과 같은 룰을 따라야하며, 추후 변동사항이 있을 수 있으니 이 점 주의하시기 바랍니다.

1. Json파일의 이름은 모두 해당 Scene의 이름과 같아야합니다.
	
	따라서, Chapter0의 대화 목록은 Chapter0.json 파일에 존재해야 합니다.

	이는 게임이 실행될때, DialogueManager가 모든 대화 내용을 로드하지 않기 때문입니다. 원한다면 그렇게 만들 수 있지만, 메모리 효율을 위해 현재는 그렇게 구성하지 않았습니다.
	대화 텍스트의 양이 정해져 있지 않으므로...


2. 모든 NPC는 NPCManager를 컴포넌트로 가지고 있으며, 고유의 ID값을 가지고 있어야합니다.
	
	Player가 상호작용을 하였을때, NPCManager 스크립트를 불러오므로 만약 NPCManger가 없다면 오류가 발생합니다.


3. Json 을 작성할때, 다른 파일의 내용을 참고하여 작성하여 주세요.

	각 구성을 설명하자면,
	spritePath는 감정표현을 위한 스프라이트 경로를 나타냅니다.

	NPC는 배열 형태로 각 NPC들의 객체를 나타냅니다.

		ID 는 배열 형태로 저장된 각 NPC들을 식별할 고유 값입니다. 이 값을 통해 sprite의 위치를 알 수 있으며, Dictionary에서 키값으로 사용되어 검색 오버헤드를 줄입니다.

		NPCName은 NPC의 이름을 나타냅니다.

		storyLine은 배열 형태로 NPC가 가진 여러 대화문 객체를 나타냅니다.

			lineID는 현재 대화문을 식별하는 값입니다. 대화문의 흐름도가 순차적이라면 배열 인덱스로도 충분히 할 수 있지만, 추후 어떤 변경사항이 있을지 모르니 추가하였습니다.

			content는 string 배열로 각 대화문의 대화들의 집합입니다.
			문장에 []을 사용하여 감정 옵션을 부여할 수 있습니다. 현재는 스프라이트를 변경하는 기능만 있습니다. 추후 이펙트도 추가할 예정입니다.
